* AWS (Amazon Web Services)

1. 정의

AWS는 아마존닷컴이 제공하는 각종 원격 컴퓨팅 서비스(클라우드)를 의미한다. 대부분의 서비스는 최종 사용자에게 직접적으로 공개되는 것이 아니고, 다른 개발자가 사용 가능한 기능을 제공하는 플랫폼을 제공하는 PaaS(Platform As a Service)이다. 아마존 웹 서비스의 각종 서비스는 REST 프로토콜 및 SOAP 프로토콜을 통해 접근, 이용 및 관리가 가능하다. 서비스를 이용하는데 과금이 부여되고 과금은 실제 사용량에 따라서 결정된다. AWS는 16개의 리전(region)에 위치하여 서비스를 제공하고 있다. AWS는 각 리전(region)의 Endpoint를 설정한다. 아래는 한국의 경우 Endpoint이다. 다른 지역에 대해서는 AWS의 리전(region) 표를 참조하면 된다.

아래는 AWS 공식 사이트 주소이다. AWS에 대한 자세한 정보를 얻을 수 있다.

- https://aws.amazon.com/ko/

Amazon Web Services는 안정성이고 확장 가능하며 저렴한 클라우드 컴퓨팅 서비스를 제공합니다. 무료로 가입할 수 있으며 요금은 사용한 만큼 지불하면 됩니다.

2. 서비스

위에 그림은 AWS에서 제공하는 서비스의 종류이다. 다양한 서비스를 제공하는 것을 확인할 수 있다. 블록체인, 기계학습, 위성, 사물인터넷(IoT), 게임개발 등 특수하고 고급수준의 서비스도 제공한다. 이번 포스팅에서는 위의 서비스 중 컴퓨팅에 EC2에 대해서 알아볼 것이다. 각각의 서비스에 대한 자세한 설명은 AWS의 관련 카테고리 페이지에 자세하게 설명되어 있다. AWS 공식 사이트는 설치방법, 레퍼런스, 가이드등이 잘 정리되어 있다. 또한 AWS를 이용하여 서비스에 대한 설계를 할 경우 표준 아이콘을 제공한다. 아래는 아이콘을 다운받을 수 있는 AWS 공식사이트이다. 

- https://aws.amazon.com/ko/architecture/icons/

* AWS EC2 

Amazon Elastic Compute Cloud는 AWS 클라우드에서 확장식 컴퓨팅을 제공한다. 물리적인 하드웨어 장비 필요없이 빠르게 Application을 개발하고 배포할 수 있는 환경을 구성한다. 원하는 만큼의 가상 서버를 구축하고 보안 및 네트워크 구성과 스토리지 관리가 가능하다. 또한 서버 스팩에 갑작스러운 변동이 발생하게 되면 변동사항에 따라 신속하게 규모를 확장하거나 축소할 수 있어 서버 트래픽에 대한 예측 필요성을 절감할 수 있다. EC2는 다음과 같은 기능을 제공한다. (AWS 공식 사이트 참조)

1) Instance : 가상 컴퓨팅 환경

2) AMI : Amazon Machine Image로 서버에 필요한 운영체제와 여러 소프트웨어들이 적절히 구성된 상태로 제공되는 템플릿

3) 인스턴스 유형 : 인스턴스를 위한 CPU, 메모리, 스토리지, 네트워킹 용량의 여러 가지 구성 제공

4) 키 페어를 사용하여 로그인 정보 보호 (AWS는 Public Key를 사용, 사용자는 Private Key를 보관하는 방식)

5) Amazon EBS(Elastic Block Store) 볼륨을 사용해 영구 스토리지 볼륨에 데이터 저장

6) 보안 그룹(Security Group)를 사용해 인스턴스에 연결할 수 있는 프로토콜, 포트, 소스IP 범위를 지정하는 가상의 방화벽 역할을 수행

7) 탄력적 IP 운영. 동적 클라우드 컴퓨팅을 위한 고정 IPv4주소

8) 태그 : 사용자가 생성하여 Amazon EC2 리소스에 할당할 수 있는 메타데이터

9) AWS 클라우드에서는 논리적으로 격리되어 있지만, 요청 시 네트워크와 간편히 연결할 수 있는 가상 네트워크(VPC)기능 제공

* AWS EC2 구축

EC2구축하는 방법을 단계적으로 알아보겠다. 서비스의 종류에 따라 다양한 서버 스팩을 설정할 수 있지만 본 포스팅에서는 가장 기본적인 옵션으로 구성하는 방법을 소개하겠다. 아래 도식도는 구축할 최종 구조이다. 

서버 구성도를 요약하자면 Client(Internet)가 AWS에 접속하는 구조이다. AWS는 기본적인 VPC(Virtual Private Cloud)보안방식을 제공하고 내부 Instance에는 Spring Framework 기반의 Web Server가 동작하고 있고, WAS는 Tomcat으로 운영되고 있다. 하나씩 구성해보겠다. 단계에 들어가기 전 AWS 회원가입은 필수이다.

1) 지역(region) 선택

대한민국에서 운영될 서비스이기에 우측 상단에 region을 서울로 선택을 한다. 지역을 선택하지 않으면 정상적인 서비스가 불가하다.

2) AMI(Amazon Machine Image) 선택

AWS 서버의 OS를 선택하는 부분이다. SUSE, RedHat, Ubuntu등 다양한 Linux OS가 있다. 운영하고자 하는 OS를 선택하면된다. 본 포스팅에서는 프리티어 계정으로 진행할 수 있는 Ubuntu OS를 선택하였다. 위의 그림에서 보면 데이터베이스 인스턴스인 Amazon RDS를 소개하고 있는데 이는 차후 Amazon RDS 관련 포스팅에서 알아보겠다.

3) 인스턴스 유형 선택

기본으로 선택되어 있는 t2.micro를 선택해도 좋고 자신의 운영환경, 서비스에 맞는 인스턴스 유형을 선택하면 된다.

4) 인스턴스 시작 검토

선택한 AMI, 인스턴스 유형에 대해서 검토 및 수정을 할 수 있다. 큰 구분으로 AMI 세부정보, 인스턴스 유형, 보안 그룹, 인스턴스 세부 정보, 스토리지, 태그가 있다. 해당 단계에서 Instance에 대한 전체적인 수정이 가능하지만 생성 후 대시보드에서도 수정가능하므로 기본 옵션으로 구축하고, 추후 운영 시 변경을 하면 된다. 본 단계에서는 스토리지 용량만 변경을 한다.

5) 스토리지 추가

위의 4) 과정에서 "스토리지 편집" 버튼을 선택한다. 서비스, 운영상의 규모에 따라 적절히 산정하여 적용한다.

6) 키 페어 생성

AWS EC2 구축단계 중 중요한 단계이다. AWS EC2의 로그인 정보 보호는 키 페어를 사용한다. AWS가 Public Key를 가지고 있고, 사용자는 Private Key를 가지고 있는 구조이다. 해당 인증 키 파일이 있어야 SSH접속을 하여 AWS 서비스를 사용할 수 있다. 처음 구성하는 것이기에 "새 키 페어 생성" 을 선택한다. 선택 후 이름을 입력하고 다운로드를 하여 사용자 컴퓨터에 안전한 곳에 위치시킨다. 

7) Instance 생성 및 대시보드

Instance를 생성 시 위와 같은 대시보드 형태로 노출되고 관리되어진다. 아래 설명 탭에 중요한 정보들이 존재한다. 인스턴스 ID, 인스턴스 상태, 퍼블릭 DNS, IPv4 퍼블릭 IP, 프라이빗 DNS, 프라이빗 IP등을 알 수 있다. 만약 인스턴스의 상태가 정상적인 running이 아니라면 구축단계를 다시 검토해야 한다. 위에 버튼들 중 작업 버튼을 이용해서 인스턴스를 시작, 재부팅등을 시킬 수 있고, 설정을 할 수 있다. 대시보드에 대한 세부정보들은 공식 사이트의 설명을 참조한다.

8) 보안그룹(Security Group) 설정

보안그룹 설정은 AWS 구축 단계 중 키 페어 설정과 더불어 중요한 단계이다. 보안 그룹은 일종의 가상의 방화벽을 설정하는것과 같은 경우이다. 보안 그룹 설정을 하기 위해서 네트워크 보안 메뉴의 보안 그룹을 선택한다. 선택을 하면 위와 같이 현재 Instance에 대한 보안 그룹이 나온다. 디폴트 설정으로 되어 있으며, 필요 시 운영자가 직접 룰(Rule)을 추가해주어야 한다. 적용할 인스턴스의 보안그룹을 선택하면 아래 표로 유형, 프로토콜, 포트 범위, 소스, 설명이 나온다. 추가를 할 시 인바운드, 아웃바운드, 태그 탭을 선택 후 "편집" 버튼을 누르면 된다.

9) 보안그룹 편집 (인바운드 규칙)

규칙을 적용하고자 하는 패턴을 등록하면 된다. 유형을 선택 시 다양하게 지정할 수 있다. 모든 트래픽, 사용자 지정 TCP 규칙 등 적용하고자 하는 유형을 선택하면 된다. 본 포스팅에서는 기본적인 HTTP통신을 위한 HTTP / 80, WAS(Tomcat) 통신을 위해 사용자 지정 TCP 규칙 / 8080, SSL통신지원을 위한 HTTPS / 443을 추가하였다. 지정과정과 결과는 아래와 같다.
