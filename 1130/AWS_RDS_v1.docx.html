[Server - AWS RDS 구축 및 Oracle 연동]

이번 포스팅은 AWS RDS 구축하는 방법과 Oracle DB를 연동하는 방법을 알아보겠습니다.

* AWS RDS (Relational Database Service)

AWS RDS는 클라우드에서 관계형 데이터베이스를 쉽게 설치, 운영 및 확장할 수 있는 웹 서비스이다. RDS는 표준 관계형 데이터베이스를 위한 경제적이고 크기 조절이 가능한 용량을 제공하고, 공통 데이터베이스 관리 작업을 한다. 아래는 AWS 가이드 문서에 설명된 RDS에 대한 특징이다. 

1) 일반적인 서버를 구축하면 CPU, 메모리, 스토리지 및 IOPS(Input / Output Operation Per Second로 초당 입력/출력 작업을 나타내는 측정 단위)가 모두 같이 제공되어진다. 하지만 RDS 서버를 구축 시 일반 서버에서 같이 묶여서 제공되던 정보들이 분할되어 독립적으로 확장할 수있다. CPU, IOPS 등 각각을 쉽게 제어가능하다.

2) RDS는 백업, 소프트웨어 패치, 자동 장애 감지 및 복구를 관리하다.

3) 관리형 환경을 제공하기 위해 RDS는 DB Instance에 대해 Shell 액세스를 제공하지 않는다

4) 필요할 때 자동화된 백업을 수행하거나 고유한 백업 스냅샷을 수동으로 만들 수 있다. 백업을 사용하여 데이터베이스를 안정적이고 효율적이게 복원할 수 있다. 

5) 기본 Instance에 대한 문제 발생 시 장애 조치를 수행할 수 있는 동기식 보조 Instance에서 가용성을 높일 수 있다.

6) MySQL, MariaDB, PostgreSQL, Oracle 및 Microsoft SQL Server 등의 데이터베이스 제품을 사용할 수 있다. 각 DB Instance는 DB 엔진을 실행하게 된다. 

7) VPC(Virtual Private Cloud) 서비스를 사용해 가상 사설 클라우드에서 DB Instance를 실행할 수 있다. 

8) 보안그룹(Security Group)은 DB Instance에 대한 액세스를 제어한다. 사용자가 지정한 IP주소 범위 또는 Amazon EC2 Instance에 액세스할 수 있도록 허용하는 방법으로 제어한다.

RDS 서비스는 프리티어로 제공되지 않으므로 과금정책을 고려하여 운영하려는 서비스를 기준으로 적절한 스팩을 산정할 필요가 있다. 보다 자세한 설명은 AWS의 가이드 문서를 참고한다.

* AWS RDS 구축

기본적으로 EC2 Instance가 구축되어 서비스되고 있다는 가정하에 RDS 서비스를 구축하는 과정을 한단계 씩 알아보겠다.

최종적인 구조는 위와 같은 구조이다. 기본적으로 AWS EC2가 서비스되고 있고, RDS서비스가 EC2랑 연동되어 Client가 EC2에 접속하여 데이터베이스 정보를 조회하는 구조이다. 일반적으로 웹 서비스라고 하면 WAS와 DB가 연동된 상태에서 서비스가 제공되는 형태인데 위와 같은 구조가 AWS 웹 서비스의 일반적인 구조라고 볼 수 있다. 아래는 RDS 구축을 할 수 있는 사이트이다.

1) DB 엔진 선택

위의 과정은 DB 엔진을 선택하는 과정이다. 운영할 서비스의 DB를 선택한다. 본 포스팅에서는 Oracle로 진행을 하겠다.

2) DB 세부정보 지정

 

위의 화면은 DB 세부 정보를 지정하는 단계이다. DB 엔진 정보는 AWS에서 제공해주는 버전의 최신버전을 선택한다. 나머지 DB Instance Class, 스토리지 유형, 할당된 스토리지 정보등 과금정보와 운영할 서비스에 맞추어서 적절히 구성해준다.

 

아래 설정정보에 대한 부분은 RDS 설정과정 중 중요한 부분 중 하나이다. 식별자 이름은 Instance에 이름을 정의하는 부분이고, 아래 마스터 사용자 이름과 마스터 암호는 DB Instance에 접속하는 계정정보이다. Web Application에서 JDBC를 이용해서 접속 시 필요한 정보이다. 운영할 서비스에 맞추어서 적절히 구성해준다. 사용자 이름과 암호는 잊어먹지 않도록 주의하여 관리한다.

3) 고급 설정 구성

위의 과정은 고급 설정 구성 중 데이터베이스 옵션 부분이다. 연결될 포트정보 및 데이터베이스 이름 등 운영될 서비스에 맞도록 적절히 설정한다. 이 외에도 고급 설정에는 VPC, 서브넷 그룹을 설정할 수 있는 네트워크 보안 부분, 백업 정책을 설정하는 백업, RDS 모니터링 관련 설정을 하는 모니터링, 로그정보, 유지관리 등 다양한 옵션들을 설정 할 수 있다. 고급 설정까지 모두 완료되면 최종적으로 데이터베이스 생성 버튼을 클릭하여 데이터베이스를 생성한다.

4) DB Instance 생성

DB Instance를 생성하는데에 설정한 옵션에 따라 오래 걸릴 수도 있다. 적절히 새로고침을 하여 과정을 확인한다.

5) DB Instance 생성

생성중에서 일정시간이 지나면 정상적으로 Instance가 생성된다, 대시보드에 "사용 가능" 이라고 상태가 노출된다. 이 밖에도 설정한 클래스, 엔진, CPU, 현재 활동(연결 수), VPC, 유지 관리 등의 정보를 확인할 수 있다. "작업" 콤보박스를 이용해서 Instance를 시작하거나 종료, 재부팅을 할 수 있다. DB이름을 선택 시 Instance에 대한 상세정보를 확인할 수 있다. 현재 활동의 경우 EC2 Instance나 Sql Developer등의 툴과 연결 될 경우 숫자가 증가하여 연결된 수를 나타낸다.

6) DB Instance 상세정보 확인

Instance 정보를 선택하여 상세정보 화면이다. 중요한 정보는 Endpoint와 Port 정보이다. 해당 Endpoint와 Port정보로 RDS에 접속하게 된다. 그 밖에도 네트워킹 정보, 보안정보를 확인할 수 있다. 

7) 보안 그룹(Security Group) 설정

RDS, EC2 두 서비스는 AWS 내부(기본 VPC가 적용된)에서 운영되고 있다. 따라서 EC2에서 RDS에 데이터베이스 정보를 조회할려면 RDS로의 접속이 필요하다. 이러한 접속은 위에 6)에서 확인한 Endpoint와 포트번호의 조합으로 가능하다. 하지만 RDS는 데이터베이스 관리 서비스인만큼 보안적인 설정을 해주어야지 접속이 가능하다. 즉 보안 그룹(가상 방화벽) 설정을 해주어야 한다. 접속은 EC2에서 이루어지므로(더 크게 보면 외부 Client 단) EC2의 보안 그룹 정책 정보가 필요하다. 현재 운영되고 있고, RDS랑 연결될 EC2의 보안 그룹 ID를 확인한다. 

8) RDS 인바운드 규칙 정의

7)에서 확인한 EC2의 보안 그룹ID를 가지고 RDS의 인바운드 규칙을 추가하여 적용한다. 이렇게 구성하게 되면 RDS에서 설정한 보안 그룹 정보가 RDS에서도 그대로 적용되는 구조가 되므로 EC2에서 허용되는 규칙이 RDS에도 동일하게 사용될 수 있다. EC2에 접속만 가능하다면 RDS의 앤드포인트와 포트정보를 가지고 RDS에 접속하여 데이터베이스 정보를 조회할 수 있다.

* EC2, RDS, Web Application간의 관계

EC2에 RDS가 연결된 상태에서 Application의 운영(배포)과정을 종합해서 보면 아래 구조도와 같다.

위의 구조도는 클라우드가 아닌 일반적인 서버 구성도와 비슷한 구조이다. 중간중간 보안 그룹(방화벽)이

존재하고 운영서버와 개발서버가 있는 상태라 볼 수 있다. 서비스 규모가 커지거나 특수 서비스가 존재할 경우 서버 이중화(Instance를 2개를 만들고 로드밸런서(Elastic Load Balancing)를 이용한 라우팅)도 고려할 수 있다. 위의 구조도를 정리하면 이렇다. 최초 외부 Client 단에서 EC2로 접속을 하게 된다. 접속을 할 시 EC2의 보안 그룹(방화벽)의 Rule에 의해서 접속을 허용받는다. EC2에서는 RDS서버로의 접속을 통해 데이터를 조회하고 다시 요청한 Client에게 응답을 하게 된다. RDS에 접속하기 위해서 동일하게 RDS의 보안 그룹(방화벽)의 Rule에 의해서 접속이 가능하다. 이러한 서비스를 개발하기 위해서 개발자는 자신의 로컬, 개발서버에서 프로그램을 개발하고 이를 EC2에 배포한다. (SFTP, 배포 자동화 도구 등) DB정보 역시 개발DB에 내용을 실제 운영되고 있는 RDS서버의 DB로 Migration, Replication을 하여 적용한다.

* DB 외부 툴 (Sql Developer) 연결

오라클 DB를 이용하여 테이블을 설계하고 쿼리문을 작성할려면 Sql Developer와 같은 IDE툴이 필요하다. 처음 연결작업을 통해 RDS 정보를 입력하여 연결을 한다. 앞서 RDS 구축 시 설정했던 마스터 이름, 마스터 비밀번호, 확인하였던 Endpoint, 포트 번호 등 위에 입력란에 맞게 작성해주면 된다. 테스트 버튼을 눌러서 "성공" 이라는 결과가 나오면 정상적으로 RDS랑 연결되었다는 것이다.
